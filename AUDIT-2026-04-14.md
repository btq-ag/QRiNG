### C2: Not quantum randomness

**File**: `qring/simulator.py` lines ~52-75
**Problem**: `_generate_quantum_bitstrings()` uses `np.random` (Mersenne Twister PRNG). Claims of "quantum measurement" and "entanglement correlations" are misleading. The correlation model `prob = quantum_bias + 0.15 * sin(k*pi/4) * (2*b_{k-1} - 1)` is an ad-hoc classical Markov chain.
**Fix options**:
1. **Best**: Integrate with Qiskit Aer. Create a quantum circuit with Hadamard gates and measurements:
```python
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator

def generateQuantumBitstring(length: int) -> list[int]:
    qc = QuantumCircuit(length, length)
    qc.h(range(length))
    qc.measure(range(length), range(length))
    sim = AerSimulator()
    result = sim.run(qc, shots=1).result()
    bits = list(result.get_counts().keys())[0]
    return [int(b) for b in bits]
```
2. **Minimum**: Keep np.random but rename to `_generateClassicalBitstrings()` and remove all "quantum" and "entanglement" claims from comments. Add a `useQuantumBackend: bool` flag.

**Acceptance criteria**: If using Qiskit, the bitstrings should pass NIST frequency test (proportion of 0s and 1s within expected range for $n$ bits).

### C3: Global RNG state mutation

**File**: `qring/simulator.py` line 36
**Problem**: `np.random.seed(seed)` sets the global NumPy RNG, affecting all callers including visualization functions.
**Fix**: Replace with `self.rng = np.random.default_rng(seed)` and use `self.rng` throughout the class. All `np.random.random()` calls become `self.rng.random()`, all `np.random.randint()` become `self.rng.integers()`.

### C4: Solidity: no access control on addNewString()

```solidity
}
```

### C5: Solidity: admin hijack via setAddresses()

**File**: `contracts/originalQRiNG.sol` lines ~42-57
**Problem**: Any caller becomes `admin`. Can be called multiple times. No initialization guard.
**Fix**: Set admin in constructor. Add guard against re-initialization:

```solidity
bool private initialized;

constructor() {
    admin = msg.sender;
}

function setAddresses(address[] memory addresses) public onlyAdmin {
    require(!initialized, "Already initialized");
    initialized = true;
    // existing logic (remove admin = msg.sender line)
}
```

### C6: Solidity: hardcoded uint[6] return

**File**: `contracts/originalQRiNG.sol` lines ~104-117
**Problem**: `randomNumber()` hardcodes `uint[6]` return size. Breaks for bitstring lengths != 6.
**Fix**: Use dynamic array:

```solidity
function randomNumber() public view returns (uint[] memory) {
    uint bitstringLength = voters[0].bitstring.length;
    uint[] memory newBitstring = new uint[](bitstringLength);
    for (uint x = 0; x < bitstringLength; x++) {
        for (uint i = 0; i < voters.length; i++) {
            newBitstring[x] ^= voters[i].bitstring[x];
        }
    }
    return newBitstring;
}
```

Note: also changed `+= / %= 2` to `^=` for gas efficiency (H5).

## High Fixes

### H1: Consensus threshold too weak

**File**: `qring/emulator.py` lines ~165-170
**Problem**: Threshold = `bitstringLength // 2`. For 6-bit strings, P(match >= 4 of 6) = 34% for random bitstrings. Not sufficient for BFT.
**Fix**: Increase threshold or increase bitstring length. For 128-bit strings, threshold at 75% gives negligible false-positive rate. At minimum, make threshold configurable and document the security analysis.
