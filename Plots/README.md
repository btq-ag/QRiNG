## Generation Instructions

### Automatic Generation

All visualizations are automatically generated when running the main scripts:

```powershell
# Generate network and quantum analysis plots
python .\Code\simulatorQRiNG.py

# Generate smart contract execution plot
python .\Code\emulatorQRiNG.py

# Generate all animations
python .\Code\visualizationQRiNG.py
```

### Custom Generation

**Specific Visualizations:**
```python
# Custom network visualization
from Code.simulatorQRiNG import QRiNGSimulator, create_qring_network_visualization
simulator = QRiNGSimulator(num_nodes=8, bitstring_length=12)
simulator.run_full_simulation()
create_qring_network_visualization(simulator, "custom_network.png")

# Custom animation parameters
from Code.visualizationQRiNG import QRiNGVisualizer
visualizer = QRiNGVisualizer()
visualizer.animate_qkd_process("custom_qkd.gif", frames=120, fps=10)
```

## Interpretation Guide

### Reading Network Plots

**Node Analysis:**
- **Size**: Larger nodes indicate higher vote counts
- **Color**: Green = honest, Yellow = suspicious, Red = dishonest
- **Labels**: Show node ID and vote count (e.g., "N0 (3)")
- **Connections**: Lines show quantum channels between nodes

**Consensus Interpretation:**
- **High Similarity**: Many green nodes indicate strong consensus
- **Mixed Results**: Yellow/red nodes suggest network disagreement
- **Threshold Analysis**: Compare vote counts to network size/2

### Understanding Quantum Plots

**Amplitude Evolution:**
- **Real Component**: Solid lines showing quantum state real part
- **Imaginary Component**: Dashed lines showing imaginary part
- **Measurement Time**: Vertical red line indicates measurement moment
- **Decoherence**: Exponential decay envelope shows quantum decoherence

**Bitstring Analysis:**
- **Similarity Matrix**: Heatmap showing pairwise node similarity
- **Color Scale**: Green = high similarity, Red = low similarity
- **Threshold Line**: Indicates consensus validation threshold

### Smart Contract Analysis

**Execution Timeline:**
- **Function Calls**: Each bar represents a smart contract function
- **Gas Usage**: Bar height indicates computational cost
- **Success/Failure**: Color coding shows transaction status
- **Cumulative Metrics**: Running totals for gas and time

**State Evolution:**
- **Variable Changes**: Track contract state over time
- **Event Emissions**: Timeline of emitted blockchain events
- **Access Control**: Admin vs. user function calls
